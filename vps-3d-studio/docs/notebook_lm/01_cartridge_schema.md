# 01: The "Cartridge" Schema Definition

This document defines the **SignConfig** interface, which serves as the "Cartridge"—a single, serializable JSON object that contains the entire state of a sign design. This object is the source of truth for the 3D renderer, the engineering engine, and the pricing service.

## Core Interface: `SignConfig`

The `SignConfig` object is structured to be easily stored in a Supabase `JSONB` column.

```typescript
export interface SignConfig {
  type: SignType;               // CHANNEL_LETTER | CABINET
  lines: string[];              // Array of text lines (up to 3)
  alignment: TextAlignment;     // LEFT | CENTER | RIGHT
  dimensions: {
    height: number;             // Letter height in inches
    depth: number;              // Return depth in inches
    calculatedWidth: number;    // Total width (derived from font metrics)
    racewayOffset: number;      // Vertical adjustment for raceway
    racewayLengths: number[];   // Manual overrides for raceway length
    racewayLengthsAuto: boolean[]; // Toggle for auto-calculation
    standoff: number;           // Distance from wall
    letterStandoff: number;     // Distance from backer to letter (for Halo)
    lineSpacing: number;        // Gap between lines in inches
  };
  illumination: IlluminationType; // FRONT_LIT | HALO_LIT | DUAL_LIT | NON_ILLUMINATED
  ledColor: LEDColor;           // WHITE | RED | BLUE | GREEN | YELLOW
  mount: MountType;             // DIRECT | RACEWAY | BACKER
  wallTexture: WallTexture;     // Visual background in 3D
  wallColor: string;            // Hex code for wall paint
  
  backerLit: boolean;           // Does the backer panel have its own halo?
  backerShape: BackerShape;     // RECTANGLE | CONTOUR
  backerPadding: number;        // Margin around letters for backer

  colors: {
    vinylSeries: string;        // Brand/Series of vinyl
    face: string;               // Face/Vinyl color name
    return: string;             // Return coil color
    trimCap: string;            // Trim cap color
    raceway: string;            // Paint color for raceway
    backer: string;             // Paint color for backer
  };
  fontFamily: string;           // ID of the font in the library
  
  faceMaterial: FaceMaterial;   // ACRYLIC | POLYCARBONATE
  faceThickness: number;        // 0.1875 (3/16") or 0.25 (1/4")
  backerMaterial: BackerMaterial; // ACM | ACRYLIC | POLYCARBONATE
  backerThickness: number;      // Thickness of the back plate
}
```

## Key Enumerations

Understanding these enums is critical for writing valid Supabase queries or filters.

### Illumination Types
- `Front Lit`: Light shines through the face.
- `Halo Lit`: Light shines out the back onto the wall/backer.
- `Front & Halo Lit`: Light shines both ways (Dual Lit).
- `Non-Illuminated`: No LEDs.

### Mounting Types
- `Direct / Flush Mount`: Letters mounted directly to the wall.
- `Raceway`: Letters mounted to a metal box containing wiring.
- `Backer Panel`: Letters mounted to a secondary panel (Rectangle or Contour).

### Materials
- `ACM`: Aluminum Composite Material (Standard for backers).
- `Acrylic`: Standard for translucent faces.
- `Polycarbonate`: High-impact alternative for faces.

## Usage in NotebookLM
When interacting with this data in Supabase:
1. **Filtering:** You can query the `cartridge_data` column using PostgreSQL JSON operators. 
   *Example:* `SELECT * FROM quotes WHERE cartridge_data->'colors'->>'face' = '031 Red';`
2. **Validation:** Ensure any JSON generated by an AI or external tool matches this structure exactly to prevent the 3D viewer from crashing.
3. **Defaults:** Refer to `constants.ts` for the standard starting values used when a new sign is created.
